# 3D Elements Thinking Guide

> 在设计 3D Web UI 时需要考虑的关键问题。

---

## 何时使用 3D Elements 风格

### ✅ 适合使用

- [ ] 页面需要突出科技感、创新力或高价值感
- [ ] 是产品的首页、落地页或展示页
- [ ] 需要强烈的视觉记忆点
- [ ] 用户群体偏好现代、高端的设计风格
- [ ] 页面信息密度适中，有足够空间展示 3D 效果

### ❌ 不适合使用

- [ ] 信息极度密集的后台管理界面
- [ ] 强调纯效率和速度的工具型页面
- [ ] 用户群体偏好简洁、极简风格
- [ ] 需要在低端设备上流畅运行
- [ ] 可访问性要求极高（3D 效果可能影响屏幕阅读器）

---

## 设计前的思考清单

### 1. 用户体验优先级

- [ ] 3D 效果是否提升了用户体验，还是只是装饰？
- [ ] 核心功能是否因为 3D 效果而更难使用？
- [ ] 用户是否能在 3 秒内理解页面的主要功能？

### 2. 性能考量

- [ ] 3D 效果的复杂度是否在可接受范围内？
- [ ] 是否在低端设备上测试过？
- [ ] 动画时长是否控制在 0.3-0.6 秒？
- [ ] 是否避免了过度使用 `backdrop-blur`？

### 3. 一致性

- [ ] 3D 风格是否与现有设计系统（iOS 风格）融合？
- [ ] 材质和质感是否在整个应用中保持一致？
- [ ] 颜色方案是否协调（冷色为主，暖色点缀）？

### 4. 可访问性

- [ ] 3D 效果是否有 fallback（禁用动画时）？
- [ ] 颜色对比度是否满足 WCAG 标准（≥ 4.5:1）？
- [ ] 动画是否可以被用户关闭？
- [ ] 焦点状态在 3D 元素上是否清晰可见？

## 常见设计决策

### 决策 1：卡片深度层次

**问题**：卡片应该有多少层深度？

**考虑因素**：
- 信息架构：前景 > 中景 > 背景
- 重要性：关键卡片更靠近用户
- 交互频率：常交互的元素更突出

**建议**：
```
前景 (translateZ-4)：主要 CTA、关键数据
中景 (translateZ-2)：普通卡片、表单
背景 (translateZ-0)：装饰元素、次要信息
```

### 决策 2：悬停效果强度

**问题**：悬停时卡片应该移动多少？

**考虑因素**：
- 移动距离：太大可能让用户感到眩晕
- 阴影变化：太小没有 3D 效果
- 性能：复杂动画影响帧率

**建议**：
```
轻微：translateY(-4px) + shadow 增强
中等：translateY(-8px) + rotateX(2deg)
强烈：translateY(-12px) + rotateX(5deg) + 发光
```

### 决策 3：材质选择

**问题**：不同元素应该使用什么材质？

**考虑因素**：
- 语义：玻璃适合信息展示，金属适合操作
- 层次：前景用实体，背景用半透明
- 一致性：同一区域材质统一

**建议**：
```
玻璃：卡片、模态框、侧边栏
金属：图标、按钮边框、装饰
霓虹：CTA 按钮、重要数据、警告
```

## 实现前的验证

### 视觉验证

- [ ] 在不同屏幕尺寸下 3D 效果是否正常？
- [ ] 在深色和浅色背景下是否都清晰？
- [ ] 动画是否流畅（无卡顿）？
- [ ] 材质是否在不同光照下效果一致？

### 交互验证

- [ ] 悬停效果是否在 0.3 秒内响应？
- [ ] 点击反馈是否明确？
- [ ] 触摸设备上是否有替代交互方式？
- [ ] 键盘导航是否正常工作？

### 代码验证

- [ ] 是否使用了 Tailwind 工具类而非内联样式？
- [ ] 是否避免了 `!important`？
- [ ] 是否使用了 CSS 变量管理主题色？
- [ ] 是否在生产环境测试过性能？

## 调试技巧

### 常见问题

1. **3D 效果不显示**
   - 检查父元素是否有 `perspective` 属性
   - 检查是否被 `overflow: hidden` 裁剪
   - 检查 `transform-style` 是否设置为 `preserve-3d`

2. **动画卡顿**
   - 减少同时动画的元素数量
   - 使用 `will-change: transform` 优化
   - 避免在动画中修改 layout 属性

3. **颜色对比度不足**
   - 使用 [WebAIM Contrast Checker](https://webaim.org/resources/contrastchecker/)
   - 调整背景或前景颜色
   - 添加描边或阴影增强对比

---

## 最佳实践总结

1. **少即是多**：3D 效果应该是点缀，不是主角
2. **性能优先**：60fps 是底线，30fps 是失败
3. **一致性**：整个应用的 3D 风格要统一
4. **可访问性**：确保所有用户都能使用
5. **测试**：在多种设备和浏览器上测试

---

**核心原则**：3D 效果是为了增强用户体验，而不是分散注意力。如果一个 3D 效果不能在 1 秒内解释清楚它的作用，那就不要用它。
